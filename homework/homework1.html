  1. Problem: Write a code that will return the machine precision for your computer (or any other computer). Give the method a name that is descriptive - for example, maceps(), or something like that. The routine should return the default machine precision. Make sure that your code is fully documented with you as the author and so on. An example of a Fortran code is given in the notes. You can translate this into a C or C++ code to do the work.

  3. Problem: To complete this problem, you may need to go to the Engineering Computer Lab on the third floor of the Engineering Building. If you have Cygwin installed on a laptop or desktop computer that you own, you can do this homework at home. Note that if you have some version of Linux/Unix on your computer, you can also do this homework using your own computer. Finally, if you are working on a MAC/Apple, you can also complete this problem using your laptop or desktop. Complete the following steps:

    1. Log onto the computer (Engineering Lab) and open a command terminal to work in.
    2. Upload the routines that you created in the first problem.
    3. Compile the routines into object modules (.o files). For example, put the files you have uploaded into a folder, say hw1_prob3, using the command 
                     % mkdir hw1_prob3
      in a Cygwin/Linux/Unix operating system. Then use 
                     % mv *.f hw1_prob3
               % cd hw1_prob3
      to move all files ending with a .f suffix to the folder just created and change the current folder to the folder just created. Finally, compile the files using 
                     % gfortran -c *.f
      or 
                     % gcc -c *.o
      using a C-compiler. The result will be a bunch of object files with a suffix of ".o".
    4. The last step in this problem is to create a shared library from the routines you have created. 
                     % ar rcv mylib *.o
      or 
                     % ar rcv mylib *.o
               % ranlib mylib.a
