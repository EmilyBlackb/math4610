  1. Problem: Write a piece of code that implements the power method for the computation of the largest eigenvalue (in magnitude) for a square matrix. Make sure that you produce an entry in your software manual. As an example, use the method for the production of a square matrix from the last assignment.
  2. Problem: Write a piece of code that implements the inverse power method for the computation of the largest eigenvalue (in magnitude) for a square matrix. Make sure that you produce an entry in your software manual. As an example, use the method for the production of a square matrix from the last assignment.
  3. Problem: Write code that uses the code from the previous two problems to estimate the 2-condition number of a matrix. Put this code in your software manual with an example using the matrix generation code from the previous assignment.
  4. Problem: Write an OpenMP version of the power method for the computation of the largest eigenvalue of a matrix.
  5. Problem: Write an OpenMP version of the inverse power method for the computation of the largest eigenvalue of a matrix.
  6. Problem: Write a code that will compute the 2-condition number of a matrix using your OpenMP code for the power method and inverse power method.
  7. Problem: Compare the 2-condition number codes with and without the OpenMP directives. Which is faster?
  8. Problem: Compare the methods in this assignment on linear systems of equations of size $40\times40$, $80\times80$, $160\times 160$, $320\times 320$, and $640\times 640$. That is, create matrices and right hand sides to match as suggest in the problems above. Compare the methods in terms of number of iterations and cpu time. Produce a graph that visualizes the work and describe what the results tell us about the various methods.
